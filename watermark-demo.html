‚ú® What is Upload-Time Watermarking?
Upload-time watermarking allows you to add text watermarks to your images during the upload process, before they are stored in GitHub. This means the watermark becomes a permanent part of the image file, making it ideal for branding, copyright protection, and professional image management.
üöÄ How It Works
  1. Select Image: Choose an image file from your device
  2. Configure Watermark: Set watermark text, color, size, opacity, position, and margin
  3. Upload: The system processes the image with the watermark applied
  4. Store: The watermarked image is saved to GitHub with metadata
  5. Access: Use the CDN URL to access your watermarked image anywhere
‚öôÔ∏è Watermark Options
You can customize every aspect of your watermark:
  • Text: Custom watermark text (company name, copyright, etc.)
  • Color: Choose any color for your watermark text
  • Size: Adjust text size from 12px to 72px
  • Opacity: Control transparency from 10% to 100%
  • Position: Place watermark at top-left, top-right, center, bottom-left, or bottom-right
  • Margin: Set distance from edges (5px to 100px)
üí° Use Cases
Upload-time watermarking is perfect for:
  • Brand Protection: Add company logos or names to prevent unauthorized use
  • Copyright: Include copyright notices on your images
  • Professional Presentation: Add watermarks for client presentations
  • Content Marketing: Brand images shared on social media
  • Portfolio Management: Mark your creative work
üîß Technical Implementation
The watermarking process uses the Jimp library to:
  • Process images in memory during upload
  • Apply text watermarks with custom styling
  • Maintain image quality while adding watermarks
  • Store both original and watermarked metadata

üîÑ Backend Processing
The backend processes watermarks before storing images:
// Check if watermark should be applied const watermarkOptions = watermarkProcessor.parseWatermarkOptions(req.body); const hasWatermark = watermarkOptions.text && watermarkOptions.text.trim() !== ''; // Apply watermark if specified if (hasWatermark) { finalBuffer = await watermarkProcessor.addWatermark(buffer, watermarkOptions); finalMetadata = await sharp(finalBuffer).metadata(); }
üí° Pro Tip: Upload-time watermarking is more efficient than dynamic watermarking because:
  • Watermarks are applied once during upload
  • No need to process images on every request
  • Better performance for frequently accessed images
  • Watermarks are permanently embedded in the file
üéØ Getting Started
To use upload-time watermarking:
  1. Navigate to any repository in your dashboard
  2. Click the "Upload Image" button
  3. Select an image file
  4. Click "Show Options" in the Watermark Options section
  5. Configure your watermark settings
  6. Click "Upload Image" to apply the watermark and upload
üîç Viewing Results
